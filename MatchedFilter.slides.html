En esta prctica usaremos:

import numpy as np

Generación de la señal de prueba¶

La idea es comparar la performance de un filtro cualquiera (LP, BP, o HP) contra el filtro adaptado. Para ello generaremos una seal que podamos procesar con ambos.

Conformaremos la seal con una parte conocida (header) y otra parte totalmente aleatoria que hace las veces de informacin til (payload).

El header sern 16 bytes 0xAC (10101100):

In [2]:
header = [1, 0, 1, 0, 1, 1, 0, 0] * 16

Los bits de informacin los generaremos aleatoriamente para simular informacin til, aunque utilizaremos una semilla determinada para poder repetir los experimentos.

In [3]:
np.random.seed(42)
info = np.random.randint(0, 2, 1500)

Finalmente, los bits de la seal completa son:

In [4]:
bits = np.concatenate((header, info))

Modulación y efecto del canal¶

La modulacin constar simplemente en envar un pulso positivo (1) o negativo (-1) segn los bis sean 0 1 respectivamente.

In [5]:
data = np.where(bits != 0, 1, -1)

Ahora consideremos los efectos del canal...

Lo primero es considerar que el canal posee mtilpes trayectorias por lo que a nuestro "receptor" llegaran N copias de la seal retardadas, por ejemplo podra ser el siguiente caso:

Para ello, en este caso sencillo modelaremos al canal como una suma de 20 copias de cada smbolo retrasadas en una muestra, con una amplitud decreciente exponencial. Es decir entre smbolo y smbolo tendremos 20 ecos (conocidos como TAPs del canal).

In [6]:
tpulse = np.linspace(-1, 1, 2*10, endpoint=False)
pulse = sp.exponential(40)[20::]
plt.figure('Pulso (respuesta del canal)')
_ = plt.stem(pulse)

Ahora armaremos la seal con el efecto del multipath:

In [7]:
signal = np.zeros(len(data)*len(pulse))
signal[10::20] = data[:]  # Se agrega un offset para que luego de la convolucin, el primer pulso comience en 0
signal = np.convolve(signal, pulse, mode='same')

Lo que, por ejemplo, para el primer byte da:

_ = plt.plot(signal[:160])

Ahra bien, para terminar de simular los efectos de un canal real y el hardware de un receptor se agrega ruido AWG y un offset de continua:

Filtrado de la señal¶

Filtro LP:¶

Definimos un filtro LP genrico en funcin del espectro del pulso envado:

In [11]:
_ = plt.stem(abs(np.fft.fft(pulse))[0:len(pulse)//2])

Del espectro de la figura anterior, un posible criterio sera filtrar a partir de la frecuencia en que los coeficientes caen por debajo del valor medio entre el mximo y el mnimo ($\sim 0.65$). Lo que aproximadamente dara una frecuencia de corte de $0.3 \cdot f_{nyq}$.

In [12]:
lpFilter = sp.firwin(len(pulse), 0.3, window= "boxcar")

Filtrando la seal con dicho filtro tenemos:

In [13]:
filteredSignalApprox = np.convolve(lpFilter, signal, mode='same')
_ = plt.plot(filteredSignalApprox[0:20*8])

Ahora, deberemos submestrear, para ello debemos "resincronizarnos" con el resultado de la convolucin. Del grfico podemos ver que el primer pulso no est en cero, sino que el pulso est en el ndice:

In [14]:
startSample = filteredSignalApprox[:20].argmax()
print("ndice inicial:{0!s}".format(startSample))

ndice inicial:4

Por lo que la seal submuestreada se obtiene comenzando por el ndice obtenido

In [15]:
filteredDataApprox = filteredSignalApprox[startSample::len(pulse)]

Finalmente, ahora debemos decidir si cada valor es un bit 1 o bit 0. Un criterio, por ejemplo, sera elegir un umbral segn lo que vemos en el grfico de aproximadamente $\sim 1.5$.

In [16]:
filteredBitsApprox = np.where(filteredDataApprox > 1.5, 1, 0)

Por ltimo para esta seccin, podemos estimar la tasa de bits errados utilizando el header:

In [17]:
BER_LP = (header != filteredBitsApprox[:8*16]).sum()/len(header)
print("Bit Error Rate: {0!s}".format(BER_LP))

Bit Error Rate: 0.3046875

Filtro adaptado¶

Para obtener la solucin ptima sabemos que el mejor filtro es aquel cuya respuesta impulsiva es idntica a la de la seal invertida en el tiempo. Por ello, podemos filtrar directamente la seal haciendo:

In [18]:
filteredSignal = np.convolve(pulse[::-1], signal, mode='same')
_ = plt.plot(filteredSignal[0:20*8])

Aunque parezca ms "ruidosa", el valor de la seal en el lugar donde el filtro est cargado con una copia del pulso, tiene un valor ms exacto que el filtro anterior. De hecho, para el ruido insertado, es el ms exacto que se puede obtener.

Ahora debemos submuestrear (de nuevo teniendo en cuenta "resincronizarnos"):

In [19]:
filteredData = filteredSignal[int(len(pulse)//2)+1::len(pulse)]

Finalmente debemos decidir por 1 0. Pero ahora usaremos lo que sabemos de teora de deteccin para definir el umbral ptimo.

Sabemos que ste se da en el punto donde las funciones de distribucin de probabilidad se cruzan. Dado que en este caso ambos bits son equiprobables y el ruido es el mismo para ambos, podemos estimar este nivel simplemente calculando la media de los valores recibidos ("centro de masa"):

In [20]:
filteredBits = np.where(filteredData > filteredData.mean(), 1, 0)

Finalmente la tasa de bits errados en el header es:

In [21]:
BER_MF = (header != filteredBits[:8*16]).sum()/len(header)
print("Bit Error Rate: {0!s}".format(BER_MF))

Bit Error Rate: 0.0859375

Lo que claramente produce una menor tasa de bits errados.

Comparación entre señales:¶

Dado que disponemos de ambas seales, podemos ver por ejemplo en cuanto difieren:

bitsDistintos = (filteredBitsApprox != filteredBits).sum()

Y dado que en este caso conocemos la seal original podemos comparar las 3:

Finalmente resulta evidente la menor tasa de bits errados con los mtodos ptimos de deteccin estudiados.
